## Philosophy

ABSUBEST is a **meta-optimizer framework**: it does not optimize within a fixed paradigm — it dynamically selects, configures, and executes the optimal optimization process for *any explicitly articulated purpose*.

The framework is built on **three irreducible claims**:

| Claim | Meaning |
|-------|---------|
| **Structural completeness** | All known failure modes of predecessor frameworks (optibest lineage) are addressed by patches P1–P14 |
| **Comparative design quality** | Best-known characterized alternative for the purpose, under current knowledge horizon K, as of deployment date |
| **Aspirational stance** | "Absolute Best" is a regulative ideal (Stratum III, Kantian sense) — a compass, not a destination |

**Key design properties:**
- **ODI-calibrated rigor** — pipeline depth adapts to task stakes (Lite / Express / Full)
- **Dual moral screens** — consequentialist (P6) AND deontological (P6b) gates
- **Counter-optimizer portfolios** — paradigm-diverse adversarial challenge (P4)
- **Bias stewardship with budget** — P5 correction budget (default 80 units, 20/correction)
- **State persistence** — checkpoint/resume across sessions (P12)
- **Cross-framework comparability** — SHA-256 comparability headers on all declarations (P13)

## Epistemic Stance

> **Tarski's Undefinability** + **Gödel's Second Incompleteness** → No framework can prove its own optimality from within.

ABSUBEST explicitly acknowledges its **Gödelian limits**:

- **Regulative Ideal**: "Absolute Best" = Stratum III orientation, not Stratum I attainment
- **Open Competitor Set**: All declarations carry expiration dates & re-verification triggers
- **Empirical Wall**: 1 internal deployment (self-application) as of v3.1
- **Indexicality**: Every declaration is bounded by context C, date D, and knowledge horizon K

Three claim types only:
1. **Structural completeness** — the framework addresses all known failure modes
2. **Comparative design quality** — best-known among characterized alternatives
3. **Aspirational stance** — orientations toward a regulative ideal

## ODI Routing

The **Optimality-Directed Investment** (ODI) score (0-10) determines pipeline rigor. Computed from 5 pre-check questions covering harm potential, scale of impact, reversibility, stakeholder diversity, and characterizability.

| ODI Range | Route | Stages | Counter-Optimization | Bias Correction | Certificate |
|-----------|-------|--------|---------------------|-----------------|-------------|
| **0-3** | **Lite** | A-F (simplified) | Optional | Optional | Heuristic / Pareto |
| **4-6** | **Express** | A-H (simplified) | Optional | Recommended | Up to statistical |
| **7-10** | **Full** | All (maximum rigor) | **Mandatory** (P4) | **Mandatory** (P5) | Any (formal preferred) |

### ODI Weight System

| Factor | Weight | Description |
|--------|--------|-------------|
| **w_I** (Irreversibility) | 5 if irreversible, else 1 | Consequences cannot be undone |
| **w_B** (Breadth/Scale) | 2 if scale >= 7, else 1 | Number and diversity of affected stakeholders |
| **w_M** (Magnitude/Harm) | 2 if harm >= 7, else 1 | Severity of potential negative outcomes |
| **w_D** (Diversity) | 1 + (n_stakeholders - 1) x 0.5 | Stakeholder count (capped) |

> **Existential-risk tasks (P0b):** w_I and w_M are **UNBOUNDED** — practitioner must justify choice >= 8 and >= 5 respectively.

## Moral Screens

ABSUBEST enforces **two independent moral screens** before any optimization begins, plus an existential-risk check.

### P6 - Consequentialist Screen (Hard Gate)

Does the stated purpose P, if achieved, risk:
1. Violating fundamental rights (life, liberty, bodily autonomy, due process)?
2. Causing severe harm to identifiable populations (discrimination, displacement, violence)?
3. Undermining democratic processes or rule of law?
4. Irreversible ecological destruction (biodiversity loss, climate tipping points)?
5. Concentrating power without accountability (surveillance, autonomous weapons, AGI control)?
6. Violating ratified international law (Geneva Conventions, UN Charter, human rights treaties)?

**Any check -> HALT. Framework returns diagnostic. No optimization performed.**

### P6b - Deontological Screen (Hard Gate, v3.1 addition)

Does the stated purpose P, if achieved:
1. Treat any persons merely as means (without consent)?
2. Require deception or manipulation of rational agents?
3. Violate a duty that holds regardless of consequences?
4. Fail the Kantian Categorical Imperative universalization test?
5. Fail the Rawlsian "veil of ignorance" test?

**Any check -> HALT. Framework returns diagnostic.**

### P0b - Existential-Risk Check (Escalation Gate, v3.1 addition)

Could failure of this task contribute to:
- Human extinction?
- Permanent civilizational collapse?
- Unrecoverable loss of moral value?
- Creation of unaligned superintelligence?

**Any check -> Formal+Redundant certification mandatory. External review board sign-off required. w_I and w_M unbounded.**

> **Appeal mechanism**: Both P6 and P6b allow Ethical Review Board override (external to framework). If the board certifies "risk mitigated by constraints C," optimization proceeds with those constraints immutable.

## Counter-Optimizer Portfolio

For ODI >= 7, ABSUBEST assembles a **paradigm-diverse counter-optimizer portfolio** Pi (P4 mandate):

### Diversity Model

```
D(Pi) = 1 - max_{i,j} sim(paradigm_i, paradigm_j)
```

Where `sim()` measures paradigm overlap across:
- **Ontological commitment** (what exists?)
- **Epistemological method** (how is knowledge established?)
- **Value orientation** (what matters?)
- **Formalization language** (how is reasoning expressed?)

### Thresholds (P11)

| D(Pi) | Tier | Action |
|-------|------|--------|
| >= 0.5 | **Tier 1** | Full portfolio -- proceed |
| 0.3-0.5 | **Tier 2** | Proceed with flagged diversity gap in declaration |
| < 0.3 | **Tier 3** | **HARD REFUSAL** -- insufficient paradigm diversity |

### Budget Allocation

| Allocation | Portion | Purpose |
|------------|---------|---------|
| Strongest current paradigm | 50% | Deep critique of current best |
| Paradigm-diverse | 30% | Cross-paradigm challenges |
| Distributed / random / human | 20% | Unanticipated failure modes |

## Transcendence Operators

When Stage F certification detects a gap, Stage G applies the appropriate **transcendence operator**:

| Operator | Trigger | Action |
|----------|---------|--------|
| **OP_COV** | Coverage gap | Expand solution space (return to Stage D) |
| **OP_DIM** | Missing dimension | Derive additional evaluation dimension (return to Stage C) |
| **OP_CON** | Convention or self-imposed constraint | Liberate constraint (return to Stage B) |
| **OP_KNO** | Knowledge gap | Acquire new knowledge (research/search) |
| **OP_FOR** | Formalization gap | Formalize previously heuristic evaluation |
| **OP_SCL** | Scale mismatch | Adjust solution scale; re-evaluate |

Each operator may cause re-entry at an earlier pipeline stage with a refined scope.


### Ceremonial Declaration

```
+-------------------------------------------------+
|  ABSUBEST CEREMONIAL DECLARATION                |
```

Declarations are archived in a SQLite database at `~/.absubest/registry/` with **P13 comparability headers** enabling cross-framework comparison.

## Hard Refusals

The pipeline **halts and returns a diagnostic** in these cases:

| Condition | Principle | Route |
|-----------|-----------|-------|
| P6 consequentialist screen fails | P6 | HALT -- diagnostic |
| P6b deontological screen fails | P6b | HALT -- diagnostic |
| P0b existential-risk without external review | P0b | HALT -- escalate |
| ODI Pre-Check incomplete | P1 | HALT -- diagnostic |
| Portfolio diversity D(Pi) < 0.3 | P11 Tier 3 | HALT -- diagnostic |
| Bias budget exhausted for ODI >= 7 | P5 | HALT -- residual distortion flag |
| State file integrity check fails | P12 | HALT -- diagnostic |

## Quick Start

### Prerequisites

- **Python >= 3.11**
- Optional: `sentence-transformers`, `scikit-learn`, `numpy`, `sqlite-utils` (for full pipeline)

### Installation

```bash
# Clone the repository
git clone https://github.com/aequchain-dev/ABSUBEST.git
cd ABSUBEST

# Install base package (requires typing-extensions)
pip install -e .

# Install with full dependencies
pip install -e ".[full]"

# Install with MCP server support
pip install -e ".[mcp]"

# Install everything (full + MCP + dev)
pip install -e ".[full,mcp,dev]"
```

### Basic Usage (Python API)

```python
from absubest import ABSUBEST

# Initialize the framework
ab = ABSUBEST(
    knowledge_horizon="2026-06-25",
    bias_steward_enabled=True,
    state_persistence=True,
)

# Optimize for a purpose
result = ab.optimize(
    purpose="Design a low-cost solar water pump for rural Senegal"
)

# View the full declaration
print(result.format())

# Access individual fields
print(f"ODI Score: {result.odi_score}")
print(f"Certificate: {result.certificate}")
print(f"Utility: {result.utility}")
print(f"Residual Risk: {result.residual_risk}")
print(f"Bias Audit: {result.bias_audit}")
```

```python
# Resume a previously checkpointed task
result = ab.resume(task_id="abc12345")
```

```bash
# Run the pipeline
```

## MCP-1 Protocol

ABSUBEST provides a **JSON-RPC 2.0 over stdio** MCP server (`absubest-mcp`), enabling integration with any MCP-capable agent environment (OpenCode, Claude Code, Cursor, etc.):

```bash
# One-shot mode (single request via stdin)
echo '{"jsonrpc":"2.0","id":"1","method":"ping","params":{}}' | absubest-mcp

# Forever mode (continuous processing)
echo '{"jsonrpc":"2.0","id":"1","method":"optimize","params":{"purpose":"test"}}' | absubest-mcp --forever
```

### MCP Methods

| Method | Params | Returns |
|--------|--------|---------|
| `ping` | `{}` | `{"pong": true, "version": "0.1.0"}` |
| `optimize` | `{"purpose": str, "context?": dict}` | Full `ABSUBESTResult` |
| `resume` | `{"task_id": str}` | Restored pipeline state -> result |
| `registry_list` | `{}` | List of archived declarations |
| `registry_get` | `{"id": str}` | Declaration by ID |
| `bias_audit` | `{"task_id": str}` | Bias audit report |
| `reverify` | `{"registry_id": str}` | Re-run verification pipeline |

### MCP Server Architecture

The MCP layer is decomposed into **6 specialized servers**:

| Server | File | Methods |
|--------|------|---------|
| **MCP-1 (Meta)** | `absubest/mcp/absubest_meta.py` | `ping`, `optimize`, `resume`, `registry_*`, `bias_audit`, `reverify` |
| **MCP-2 (Stage)** | `absubest/mcp/absubest_stage.py` | `stage_a`-`stage_h`, `run_full_pipeline` |
| **MCP-3 (Counter-Opt)** | `absubest/mcp/absubest_counter_opt.py` | `portfolio_assemble`, `portfolio_run`, `portfolio_diversity`, `portfolio_aggregate` |
| **MCP-4 (Archive)** | `absubest/mcp/absubest_archive.py` | `archive_store`, `archive_get`, `archive_search`, `archive_list`, `archive_delete`, `archive_compare` |
| **MCP-5 (State)** | `absubest/mcp/absubest_state.py` | `state_save`, `state_load`, `state_resume`, `state_list`, `state_delete`, `state_integrity` |
| **MCP-6 (Bias)** | `absubest/mcp/absubest_bias.py` | `bias_detect`, `bias_disclose`, `bias_apply_correction`, `bias_veto`, `bias_report`, `bias_self_apply` |

## OpenCode Agent Integration

ABSUBEST ships as **15 OpenCode skills** in `SKILLS/`, one per pipeline stage. Each skill is a standalone markdown file that can be loaded into any OpenCode-compatible agent environment:

```bash
# Load a specific skill
opencode skill load "SKILLS/ODI Pre-Check/SKILL.md"
opencode skill load "SKILLS/Moral Screen/SKILL.md"
opencode skill load "SKILLS/Purpose Crystalization/SKILL.md"
```

The `SUITE/` directory contains the orchestrator specification:
- **`SUITE/AGENT.md`** -- Full ABSUBEST-Exactor agent specification (decision sequence, escalation paths, sub-agent architecture)
- **`SUITE/SKILLS.md`** -- Complete specifications for S1-S15
- **`SUITE/TOOLING.md`** -- Complete specifications for Tier 1-4 tools
- **`SUITE/ABSUBEST_Agent_Suite_Tooling_Environment_Implementation_Plan.pdf`** -- Full design rationale with certification
